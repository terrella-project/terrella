## API keys (where they live)

Don't put real keys in this file. Real keys live in:

- `stack/.env` — generated by `scripts/generate-env.sh`; fill in `ANTHROPIC_API_KEY`, `GEMINI_API_KEY`, `OPENAI_API_KEY` after generation. See [setup/06-observability-stack.md](../../docs/setup/06-observability-stack.md#62-environment-variables).
- LiteLLM reads them at startup via env interpolation in [`../../stack/observability/litellm/config.yaml`](../../stack/observability/litellm/config.yaml).

After changing `stack/.env`, run `docker compose up -d litellm litellm-exporter` from `stack/`. A plain `docker compose restart` does not reload interpolated env values.

## Manual monthly billing entry

After each billing cycle:

```bash
cd ~/src/jomkz/terrella/stack/observability
./scripts/log-billing.sh
# prompts: month (YYYY-MM), vendor, amount USD, notes
```

This inserts a row into the `monthly_costs` table in Postgres. Grafana's "Total spend" panel sums those rows alongside the per-call costs from LiteLLM so all four lines (Copilot, Claude Code, Anthropic, Gemini) appear together.
